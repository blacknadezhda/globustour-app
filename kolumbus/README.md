# 🧭 Коломбус — оркестратор продаж Globus Tour

Реализация архитектуры «Коломбус × Глоба v2» (см. `/kolumbus.html` в корне репозитория).

- **Вход:** Telegram-бот (aiogram 3.x) — единственная точка входа
- **Мозг:** Claude (`claude-opus-4-8`), язык с клиентами — украинский
- **CRM:** All Inclusive API (`api.allinclusivecrm.com/v1`, Bearer token)
- **Монитор:** отдельный сторож `watchdog.py` — heartbeat каждую минуту, алерт в Telegram

## Что умеет

**Режим ПРОДАЖИ (клиенты):** отвечает 24/7 украинским языком, квалифицирует лид
(направление · даты · состав · бюджет · телефон, JTBD-тип), продаёт эмоцию, знает
направления и сезонность, и по готовности сам создаёт лид в CRM (`POST /site/api/create`)
с саммари и подсказками по апселам для менеджера + пингует чат команды.

**Железное правило зашито в промпт:** цены на туры называет только менеджер.
AI не оформляет и не продаёт.

**Режим КОМАНДА (Нико, Виктория, менеджеры — по `TEAM_USER_IDS`):** читает CRM
(лиды/висяки, платежи, менеджеры), готовит выжимки, делегирует операционку Глобе
(`delegate_to_globa` → сообщение в чат Глобы), играет тренажёр возражений
(«зіграй клієнта» → роль сложного клиента + разбор).

**Контролёр:** каждое утро (`DIGEST_TIME`, по умолчанию 09:00 Киев) сам собирает
из CRM и шлёт в чат команды РОП-дайджест: продажи, новые лиды, висяки 3+ дня,
SLA, фокус дня; 1-го числа — плюс ежемесячное ретро и материал к 1:1.
Команда `/digest` — запустить вручную.

**Вебхуки CRM (реальное время):** при заданных `WEBHOOK_PORT`/`WEBHOOK_SECRET`
поднимается HTTP-приёмник `POST /webhook/crm/<секрет>` — CRM сама сообщает о
смене статуса лида и новых платежах, Коломбус фильтрует шум и постит значимое
в чат команды. Перед боевым включением проверьте формат событий на webhook.site.

## Структура

```
kolumbus/
├── kolumbus/            # пакет бота
│   ├── main.py          # вход: aiogram-бот + heartbeat
│   ├── brain.py         # Claude + цикл инструментов (create_lead, CRM, Глоба)
│   ├── prompts.py       # системный промпт из правил + базы знаний
│   ├── crm.py           # клиент All Inclusive CRM
│   ├── scheduler.py     # контролёр: дайджест, висяки, SLA, ретро
│   ├── webhooks.py      # приёмник событий CRM (реальное время)
│   ├── globa.py         # делегирование Глобе
│   ├── sessions.py      # память диалогов (JSON)
│   └── config.py        # конфиг из .env
├── knowledge/           # ← экспертный капитал: наполняют Нико и Виктория
│   ├── destinations.yaml   # направления: для кого, фишки, JTBD
│   ├── seasonality.yaml    # месяц × направление
│   ├── emotions.yaml       # банк эмоциональных формулировок
│   └── upsells.yaml        # матрица апселов (подсказки менеджеру)
├── watchdog.py          # 🩺 сторож: только stdlib, без AI — потому надёжен
├── requirements.txt
└── .env.example
```

## Запуск

```bash
cd kolumbus
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env      # заполнить токены и ID
python -m kolumbus.main   # бот
python watchdog.py        # сторож (отдельный процесс, отдельный systemd-юнит)
```

Минимум для старта: `TELEGRAM_BOT_TOKEN` + `ANTHROPIC_API_KEY`. Без `CRM_TOKEN`
бот работает, но создание лида и дайджест вернут ошибку (Коломбус честно
скажет об этом, цифры выдумывать не будет).

## База знаний

Файлы в `knowledge/` — сердце продаж. Заполнены стартовым топ-5 направлений
как образец формата; дополняйте своими. Бот перечитывает их при рестарте.

## Будущие агенты

Дирижёр уже готов к росту: новый агент = новый инструмент в `brain.py`
(по образцу `delegate_to_globa`) — без переписывания ядра.
